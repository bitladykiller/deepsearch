# 02 - 配置系统

## 1. 配置架构概览

配置系统由两个核心类组成：

```
┌─────────────────────────────────────────────────────────┐
│                    配置优先级链                           │
│                                                         │
│   环境变量 (.env)  >  config.json  >  代码默认值         │
│         ▲                                               │
│         │                                               │
│    dotenv 自动加载                                       │
│    (项目根目录 .env)                                     │
└─────────────────────────────────────────────────────────┘

┌──────────────────────────┐    ┌──────────────────────────┐
│      AppConfig           │    │     APISettings          │
│   (核心运行时配置)        │    │   (FastAPI 服务配置)     │
│                          │    │                          │
│   · api_key              │    │   · app_name             │
│   · model                │    │   · app_env              │
│   · user_id / tenant_id  │    │   · host / port          │
│   · 记忆系统配置          │    │   · cors_allow_origins   │
│   · 向量数据库配置        │    │   · config_path          │
│   · 迭代配置              │    │                          │
└──────────────────────────┘    └──────────────────────────┘
```

### 1.1 文件位置

| 文件 | 路径 | 用途 |
|------|------|------|
| `config.py` | `src/deep_research/config.py` | 配置类定义 |
| `config.json` | 项目根目录 | 运行时配置文件 |
| `.env` | 项目根目录 | 环境变量（Git 忽略） |
| `.env.example` | 项目根目录 | 环境变量模板 |

## 2. AppConfig 详解

### 2.1 数据结构

```python
@dataclass(frozen=True)
class AppConfig:
    """核心运行时配置：模型、记忆、向量数据库等。"""
    
    # ---- LLM 配置 ----
    api_key: str                    # DashScope API Key（必填）
    model: str                      # LLM 模型名，默认 "qwen-plus"
    
    # ---- 用户标识 ----
    thread_id: str                  # 会话线程 ID，默认 "default"
    user_id: str                    # 用户 ID，默认 "default_user"
    tenant_id: str                  # 租户 ID，默认 "default_tenant"
    
    # ---- 研究流程配置 ----
    max_iterations: int             # 最大研究迭代轮数，默认 3
    
    # ---- 记忆系统配置 ----
    enable_memory: bool             # 是否启用记忆系统，默认 True
    short_term_ttl_seconds: int     # 短期记忆 TTL（秒），默认 604800（7天）
    short_term_max_messages: int    # 短期记忆最大消息数，默认 30
    short_term_summary_threshold: int  # 触发摘要的消息数阈值，默认 20
    short_term_backend: str         # 短期记忆后端，默认 "mysql"
    long_term_backend: str          # 长期记忆后端，默认 "mysql"
    long_term_scope: str            # 长期记忆作用域，默认 "user"
    save_conversation_task: bool    # 是否保存对话为任务记录，默认 False
    checkpointer_backend: str       # LangGraph checkpointer 后端，默认 "redis"
    enable_milvus: bool             # 是否启用 Milvus，默认 True
    memory_top_k: int               # 记忆检索返回数量，默认 6
    
    # ---- 基础设施连接 ----
    redis_url: str                  # Redis 连接串
    mysql_dsn: str                  # MySQL 连接串（DSN 格式）
    milvus_host: str                # Milvus 主机地址
    milvus_port: int                # Milvus 端口，默认 19530
    milvus_collection: str          # Milvus 集合名，默认 "mult_agent_memory"
```

### 2.2 frozen=True 的含义

`AppConfig` 使用 `@dataclass(frozen=True)` 装饰器，这意味着：
- 所有实例创建后不可修改（不可变对象）
- 尝试修改属性会抛出 `FrozenInstanceError`
- 线程安全：多线程环境下可以安全共享
- 哈希可用：可以作为字典的键或放入集合

### 2.3 with_overrides 方法

```python
def with_overrides(self, **kwargs) -> "AppConfig":
    cleaned = {k: v for k, v in kwargs.items() if v is not None}
    return replace(self, **cleaned)
```

由于 `frozen=True`，不能直接修改属性。`with_overrides` 使用 `dataclasses.replace` 创建一个新的 `AppConfig` 实例，只覆盖非 None 的字段。

使用场景：
- CLI 启动时覆盖部分配置
- API 请求时覆盖 user_id、thread_id 等
- 测试时注入不同的配置

示例：
```python
base_config = AppConfig.from_file()
runtime_config = base_config.with_overrides(
    user_id="user_123",
    thread_id="thread_456",
    max_iterations=5,
)
# base_config 不变，runtime_config 是新实例
```

## 3. 配置加载流程

### 3.1 from_file 流程

```
AppConfig.from_file(path)
    │
    ├── 1. 确定配置文件路径
    │   ├── 如果传入 path，使用该路径
    │   └── 否则使用项目根目录下的 config.json
    │
    ├── 2. 读取并解析 JSON
    │   ├── 检查文件是否存在
    │   ├── 解析 JSON 为 dict
    │   └── 校验格式
    │
    ├── 3. 解析 api_key
    │   ├── 先查环境变量 DASHSCOPE_API_KEY
    │   ├── 再查 config.json 中的 api_key
    │   └── 都没有则抛出 ValueError
    │
    └── 4. 调用 _from_dict 构建完整配置
        └── 逐字段解析，环境变量优先于文件值
```

### 3.2 _resolve_str 解析逻辑

```python
@staticmethod
def _resolve_str(data: dict, field: str, env_key: str, default: str = "") -> str:
    # 优先级 1：环境变量
    env_value = os.getenv(env_key)
    if env_value is not None and str(env_value).strip() != "":
        return str(env_value).strip()
    
    # 优先级 2：config.json 中的字段
    file_value = data.get(field)
    if file_value is not None and str(file_value).strip() != "":
        return str(file_value).strip()
    
    # 优先级 3：默认值
    return default
```

每个字段的解析规则：

| 字段 | 环境变量 Key | config.json Key | 默认值 |
|------|-------------|----------------|--------|
| `api_key` | `DASHSCOPE_API_KEY` | `api_key` | （必填） |
| `model` | `MODEL` | `model` | `qwen-plus` |
| `thread_id` | `THREAD_ID` | `thread_id` | `default` |
| `user_id` | `USER_ID` | `user_id` | `default_user` |
| `tenant_id` | `TENANT_ID` | `tenant_id` | `default_tenant` |
| `max_iterations` | `MAX_ITERATIONS` | `max_iterations` | `3` |
| `enable_memory` | `ENABLE_MEMORY` | `enable_memory` | `true` |
| `short_term_ttl_seconds` | `SHORT_TERM_TTL_SECONDS` | `short_term_ttl_seconds` | `604800` |
| `short_term_max_messages` | `SHORT_TERM_MAX_MESSAGES` | `short_term_max_messages` | `30` |
| `short_term_summary_threshold` | `SHORT_TERM_SUMMARY_THRESHOLD` | `short_term_summary_threshold` | `20` |
| `short_term_backend` | `SHORT_TERM_BACKEND` | `short_term_backend` | `postgres` |
| `long_term_backend` | `LONG_TERM_BACKEND` | `long_term_backend` | `postgres` |
| `long_term_scope` | `LONG_TERM_SCOPE` | `long_term_scope` | `user` |
| `save_conversation_task` | `SAVE_CONVERSATION_TASK` | `save_conversation_task` | `false` |
| `checkpointer_backend` | `CHECKPOINTER_BACKEND` | `checkpointer_backend` | `auto` |
| `enable_milvus` | `ENABLE_MILVUS` | `enable_milvus` | `true` |
| `memory_top_k` | `MEMORY_TOP_K` | `memory_top_k` | `6` |
| `redis_url` | `REDIS_URL` | `redis_url` | `redis://127.0.0.1:6379` |
| `mysql_dsn` | `MYSQL_DSN` | `mysql_dsn` | `mysql+pymysql://...` |
| `milvus_host` | `MILVUS_HOST` | `milvus_host` | `127.0.0.1` |
| `milvus_port` | `MILVUS_PORT` | `milvus_port` | `19530` |
| `milvus_collection` | `MILVUS_COLLECTION` | `milvus_collection` | `mult_agent_memory` |

### 3.3 布尔值解析

```python
@staticmethod
def _resolve_bool(data: dict, field: str, env_key: str, default: bool) -> bool:
    value = AppConfig._resolve_str(data, field, env_key, "true" if default else "false")
    return value.lower() == "true"
```

只有字符串值为 `"true"`（不区分大小写）时才返回 `True`，其他所有值（包括 `"1"`、`"yes"`）都返回 `False`。

### 3.4 整数值解析

```python
@staticmethod
def _resolve_int(data: dict, field: str, env_key: str, default: int) -> int:
    value = AppConfig._resolve_str(data, field, env_key, str(default))
    return int(value)
```

如果环境变量或配置文件中的值无法转换为整数，会抛出 `ValueError`。

## 4. APISettings 详解

### 4.1 数据结构

```python
@dataclass(frozen=True)
class APISettings:
    """FastAPI 服务专用配置。"""
    
    app_name: str = "DeepResearch Multi-Agent Assistant"  # 应用名称
    app_env: str = "development"                          # 运行环境
    host: str = "0.0.0.0"                                 # 监听地址
    port: int = 8000                                      # 监听端口
    cors_allow_origins: str = "http://localhost:5173,..."  # CORS 允许的源
    config_path: str = ""                                 # config.json 路径
```

### 4.2 __post_init__ 处理

```python
def __post_init__(self):
    if not self.config_path:
        object.__setattr__(self, "config_path", str(_PROJECT_ROOT / "config.json"))
```

由于 `frozen=True`，不能直接在 `__init__` 中赋值。使用 `object.__setattr__` 绕过冻结检查，设置默认的配置文件路径。

### 4.3 CORS 源解析

```python
def cors_origins(self) -> list[str]:
    values = [item.strip() for item in self.cors_allow_origins.split(",")]
    return [item for item in values if item]
```

将逗号分隔的字符串拆分为列表，去除空白和空字符串。

## 5. .env 文件加载

### 5.1 加载时机

```python
_PROJECT_ROOT = Path(__file__).resolve().parents[2]
_ENV_PATH = _PROJECT_ROOT / ".env"
if _ENV_PATH.exists():
    load_dotenv(_ENV_PATH)
```

配置模块在导入时就会尝试加载 `.env` 文件。这意味着：
- `.env` 中的变量在 `config.py` 被导入时就已经可用
- 之后通过 `os.getenv()` 读取的就是 `.env` 中的值
- 但 `os.environ` 中已有的变量不会被覆盖（`load_dotenv` 默认行为）

## 7. 配置使用场景

### 7.1 CLI 模式

```
python -m deep_research.cli --config config.json --user-id user_001
```

流程：
1. `parse_cli_args()` 解析命令行参数
2. `build_runtime_config(args)` 加载配置并应用覆盖
3. `AppConfig.from_file(args.config)` 读取 config.json
4. `config.with_overrides(...)` 应用 CLI 参数覆盖

### 7.2 API 模式

```
uvicorn deep_research.api.app:app
```

流程：
1. `APISettings.from_env()` 从环境变量加载 API 配置
2. `create_app()` 创建 FastAPI 应用
3. `get_workflow_service()` 懒加载 WorkflowService 单例
4. `WorkflowService.__init__` 中通过 `AppConfig.from_file()` 加载核心配置
5. 每个 API 请求时通过 `_build_runtime_config` 应用请求级别的覆盖

### 7.3 Docker 模式

Docker Compose 环境变量通过 `environment` 字段注入：
```yaml
environment:
  MYSQL_DSN: "mysql+pymysql://deepresearch:deepresearch123@mysql:3306/deepresearch?charset=utf8mb4"
  REDIS_URL: "redis://redis:6379"
  CHECKPOINTER_BACKEND: redis
  SHORT_TERM_BACKEND: mysql
  LONG_TERM_BACKEND: mysql
  MILVUS_HOST: milvus
  MILVUS_PORT: 19530
```

这些环境变量的优先级高于 config.json 中的同名字段。
